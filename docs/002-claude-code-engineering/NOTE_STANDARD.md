# 极客时间专栏学习笔记规范

> 本规范沉淀自第一篇文章（开篇词）的笔记迭代过程，用于保证后续每篇笔记的**风格一致、结构完整、信息密度达标**。

---

## 一、目录结构

```
docs/002-{course-name}/
├── README.md                  # 专栏总说明 + 学习清单
├── NOTE_STANDARD.md           # 本文件：笔记规范
├── article-origin/            # ⚠️ .gitignore，不提交
│   └── 001-{slug}/            # 原文 JSON/HTML/comments
└── 001-{slug}/                # 学习笔记
    └── README.md
```

**命名规则**：`{序号}-{简短中文描述}`，如 `001-共生而非替代`，序号三位数。

---

## 二、笔记必备章节

每篇 `README.md` 按以下顺序组织：

```markdown
# {序号} · {文章标题}
元信息（出处、时间）

## 这篇文章在回答什么问题？
[2-3 个具体问题点明文章讨论什么]

## 原文转述
[费曼式用自己的话逐段转述论证逻辑]
### 一、{核心论点}
💡 补充理解 / 类比 / 反例

## {文章框架/对照表}
[结构性的总结，作为导航图]

## 技术关键词（待深入）
[表格：关键词 | 文中表述 | 我的理解 | 后续跟进]

## 我的疑惑与待验证
[遗留问题清单]

## 评论区高价值讨论
[按话题提炼有信息量的问答]

## 相关链接
```

---

## 三、写作原则

- **费曼转述**：用自己的话重新讲，不复制原文。加类比、举例、场景化
- **💡 标记**：个人理解和补充用 `💡` 开头，与原文转述做视觉区分
- **信息密度**：不看原文也能理解核心脉络，不能只有空洞的「感觉」「共鸣」
- **问题导向**：每篇开头用具体问题说明文章在回答什么
- **遗留钩子**：每篇末尾列出没搞懂的问题，后续学习时可回来验证

---

## 四、内容边界

**不写**：课程排期/运营信息、推销话术、仪式性文字、纯打卡留言

**必须写**：核心概念定义、框架/分类/对比、关键类比示例、没搞懂的问题、评论区有信息量的讨论

---

## 五、Git 管理

- `article-origin/` 已在 `.gitignore`（`**/article-origin/`），原文数据不提交
- Commit 格式：`docs: {简短描述}`

---

## 六、评论区筛选

记录标准：作者回复且信息量高、读者问了我也想问的、提供了正文没覆盖的额外信息

忽略：纯打卡、纯赞美、无实质内容
